# Week 2 Recap

Week 2 was the edge classifier: synthetic Aido Rover sensor data, a small feedforward net, held-out eval, INT8 quantization, and a CPU latency benchmark. The thing I kept thinking about while writing this up was how different "quantization" ended up looking here compared to the 4-bit GGUF call I made on Psychologist Agent, even though on paper they're the same kind of decision.

With Psychologist Agent, quantizing Llama-3.1-8B down to GGUF Q4_K_M wasn't really a choice, more like a requirement. Fine-tuned in fp16 that model is around 16GB, and that just doesn't fit comfortably on an RTX 4060 with 8GB of VRAM alongside everything else running, and even if it barely fit, it would've been painfully slow. Q4_K_M got it down to 4.6GB and running at around 43.5 tok/s, which is what actually made local deployment possible in the first place.

This week was pretty much the opposite situation. The fp32 classifier is tiny, like 17K parameters, 0.066MB, and it was already running in 0.035ms on CPU, something like 578x under the 20ms budget before I even touched quantization. INT8 cost a small amount of accuracy at the full 4,000-sample dataset size (92.50% down to 92.33%, about 0.17 points), and it also didn't make anything faster. It actually came out slower, because dynamic quantization has to work out the activation scale at runtime every call, and on a network this small that overhead is bigger than whatever the int8 matmul saves. The only thing quantization actually bought me was a smaller file, about 68.6% smaller, and that only matters because Sentinel and Rover are apparently running several models at once on one chip, not because this classifier needed to shrink to work.

So I don't think the takeaway is "quantization helps." It's more that whether quantization does anything for you depends completely on what's actually the bottleneck. For Psychologist Agent, both memory and speed were genuinely the bottleneck, so 4-bit quantization solved a real problem. For this classifier nothing was the bottleneck to begin with, so there wasn't really anything for quantization to fix. The size reduction is real but kind of beside the point when the model was already 66KB. I probably would've just assumed quantization made it faster if I hadn't actually timed it, which is honestly the more useful thing I got out of this week.
